From HTTP 0.9 to 1.0: The Genesis and First Major Upgrade

The Genesis: HTTP 0.9 (1989)

HTTP was created by Sir Tim Berners-Lee. While working at Cern in 1989, Berners-Lee imagined a system designed to share information across a network of computers. This system would later be used To make the World Wide Web. It was a simple yet effective protocol used to request and transmit documents.

Its initial iteration, HTTP 0.9, was a simple one-line protocol consisting of a single “GET” method followed by the path of the desired document. Berners-lee stated, “I just had to take the hypertext idea and connect it to TCP and DNS ideas and ta-da! The World Wide Web”. This initial version was the critical first step, allowing documents written in HTML to be retrieved, creating the backbone of the internet.

The First Major Upgrade: HTTP 1.0

The rapid growth of the World Wide Web in the early 1990s made an upgrade a necessity. This led Berners-Lee to create HTTP 1.0 with the help of Roy Fielding and Henrik Neilsen.

HTTP 1.0 introduced several key features, transforming the protocol from a simple document retrieval tool to a transfer tool with many uses

  • **Metadata Transfer: The ability to transfer metadata about requests and responses (using headers).
  • **Media Transfer: The ability to transfer media such as images and other file types. This was crucial for the web’s multimedia features to exist.

Fielding remarked, “I knew the Web would be the printing press of the internet age. Its changing society and the way people think”.
