Key Concepts in the History of HTTP

Hypertext Transfer Protocol (HTTP)
The foundation of data communication for the World Wide Web, serving as the invisible handshake that connects web clients and servers.
Hypertext
Text displayed on a computer display with references (hyperlinks) to other text that the reader can immediately access. It is the core idea that Berners-Lee connected to networking protocols.
TCP & DNS
Berners-Lee had to connect the hypertext idea to **TCP (Transmission Control Protocol)** and **DNS (Domain Name System)** ideas to create the World Wide Web. TCP manages the reliable connection, and DNS translates domain names into IP addresses.
Metadata
Data about data. In HTTP, this is transferred via headers in requests and responses, introduced in version 1.0.
Persistent Connections
An HTTP 1.1 feature that keeps a single TCP connection open to send and receive multiple HTTP requests and responses, improving efficiency and reducing latency.
Multiplexing
The key innovation in HTTP/2, allowing multiple requests to be sent and responses to be received concurrently over a single TCP connection, overcoming the "Head-of-Line Blocking" issue of HTTP 1.1.
